LUA基本语法

发表于2023-10-16|更新于2023-10-16
|阅读量:

1. 概述

1.1 需求

目前基于计算机标准总线(如PCI,ISA总线)的运动控制器大多采用在高级编程语言环境(VC，C#等)中调用API接口函数的使用方式。为了实现运控工艺实现灵活以及脱机运行，需要开发一种通用运动控制脚本。

1.2 目标

替换运动程序 (运行效率，语法灵活性，变量限制)

1.3 选择LUA的原因

轻量级且高效的

嵌入式脚本语言，可在裸核上运行

面向原型的编程语言，语法与C类似

与C语言交互便利

2. LUA语法

2.1 数据类型

Lua 是动态类型语言，变量不要类型定义, 只需要为变量赋值。 值可以存储在变量中，作为参数传递或结果返回。

当前可用类型分别为：nil、boolean、number、string、userdata、function 和 table。

数据类型 描述
nil 只有值nil属于该类，表示一个无效值（在条件表达式中相当于false）
boolean 包含两个值：false和true
number 表示双精度类型的实浮点数，lua 5.3+版本支持64位长整型
string 字符串由一对双引号或单引号来表示
function 由 C 或 Lua 编写的函数
userdata 表示任意存储在变量中的C数据结构
table Lua 中的表（table）是一个”关联数组”（associative arrays），数组的索引可以是数字、字符串或表类型。table 的创建是通过”构造表达式”来完成，最简单构造表达式是{}，用来创建一个空表。

2.1.1 字符串（String）

字符串是由数字、字母、下划线组成的一串字符。

字符串可以使用以下三种方式来表示：

  • 单引号间的一串字符。
  • 双引号间的一串字符。
  • [[ 与 ]] 间的一串字符。

2.1.2 表（Table）

每个 Table 结构，最多会由三块连续内存构成。

  • 一个 table 结构
  • 一块存放了连续整数索引的数组
  • 和一块大小为 2 的整数次幂的哈希表

2.1.3 Userdata

2.2 运算符

运算符是一个特殊的符号，用于告诉解释器执行特定的数学或逻辑运算。Lua提供了以下几种运算符类型：

  • 算术运算符
  • 关系运算符
  • 逻辑运算符
  • 位运算符
  • 其他运算符

2.2.2 关系运算符

下表列出了 Lua 语言中的常用关系运算符，设定 A 的值为10，B 的值为 20：

操作符 描述 实例
== 等于，检测两个值是否相等，相等返回 true，否则返回 false (A == B) 为 false。
~= 不等于，检测两个值是否相等，不相等返回 true，否则返回 false (A ~= B) 为 true。
> 大于，如果左边的值大于右边的值，返回 true，否则返回 false (A > B) 为 false。
< 小于，如果左边的值大于右边的值，返回 false，否则返回 true (A < B) 为 true。
>= 大于等于，如果左边的值大于等于右边的值，返回 true，否则返回 false (A >= B) 返回 false。
<= 小于等于， 如果左边的值小于等于右边的值，返回 true，否则返回 false (A <= B) 返回 true。

2.2.3 逻辑运算符

下表列出了 Lua 语言中的常用逻辑运算符，设定 A 的值为 true，B 的值为 false：

操作符 描述 实例
and 逻辑与操作符。 若 A 为 false，则返回 A，否则返回 B。 (A and B) 为 false。
or 逻辑或操作符。 若 A 为 true，则返回 A，否则返回 B。 (A or B) 为 true。
not 逻辑非操作符。与逻辑运算结果相反，如果条件为 true，逻辑非为 false。 not(A and B) 为 true。

2.2.4 位运算符

下表列出了 Lua 语言中的常用位运算符，设定 A 的值为1，B 的值为 0：

操作符 描述 实例
& 按位与运算符。 A & B 为 0。
| 按位或运算符。 A | B 为 1。
~ 按位异或运算符。 A ~ B 为 1。
>> 右移运算符。 A >> 1 为 0。
<< 左移运算符。 A << 1 为 2。
~ 按位非运算符。 ~ A 为 0。

所有的位操作都将操作数先转换为整数， 然后按位操作，其结果是一个整数。

对于右移和左移，均用零来填补空位。移动的位数若为负，则向反方向位移；若移动的位数的绝对值大于等于整数本身的位数，其结果为零 （所有位都被移出）。

操作符 描述 实例

# 一元运算符，返回字符串或表的长度。 #”Hello” 返回 5

2.3 循环

Lua 语言提供了以下几种循环处理方式：

循环类型 描述
while 循环 在条件为 true 时，让程序重复地执行某些语句。执行语句前会先检查条件是否为 true。
for 循环 重复执行指定语句，重复次数可在 for 语句中控制。
repeat…until 重复执行循环，直到 指定的条件为真时为止
循环嵌套 可以在循环内嵌套一个或多个循环语句（while do … end; for … do … end; repeat … until;）

循环控制语句用于控制程序的流程， 以实现程序的各种结构方式。

Lua 支持以下循环控制语句：

控制语句 描述
break 语句 退出当前循环或语句，并开始脚本执行紧接着的语句。
goto 语句 将程序的控制点转移到一个标签处。

2.3.1 while循环

statements(循环体语句) 可以是一条或多条语句，condition(条件) 可以是任意表达式，在 condition(条件) 为 true 时执行循环体语句。

2.3.2 for循环

var 从 exp1 变化到 exp2，每次变化以 exp3 为步长递增 var，并执行一次 **”执行体”**。exp3 是可选的，如果不指定，默认为1。

2.3.3 repeat…until

condition(条件) 在statements(循环体语句) 末尾，每执行一次statements(循环体语句) 进行一次判断。

2.3.4 循环嵌套

Lua 编程语言中允许循环中嵌入循环。

2.4 流程控制

Lua 提供了以下控制结构语句：

语句 描述
if 语句 if 语句 由一个布尔表达式作为条件判断，其后紧跟其他语句组成。
if…else 语句 if 语句 可以与 else 语句搭配使用, 在 if 条件表达式为 false 时执行 else 语句代码。
if 嵌套语句 可以在if 或 else if中使用一个或多个 if 或 else if 语句 。

2.4.1 if 语句

在布尔表达式为 true 时会if中的代码块会被执行，在布尔表达式为 false 时，紧跟在 if 语句 end 之后的代码会被执行。

2.4.2 if…else 语句

在布尔表达式为 true 时会if中的代码块会被执行，在布尔表达式为 false 时，else 的代码块会被执行。

2.5 函数

2.5.1 函数定义

Lua 编程语言函数定义格式：

  • optional_function_scope: 该参数是可选的制定函数是全局函数还是局部函数，未设置该参数默认为全局函数，如果你需要设置函数为局部函数需要使用关键字 local。
  • function_name: 指定函数名称。
  • argument1, argument2, argument3…, argumentn: 函数参数，多个参数以逗号隔开，函数也可以不带参数。
  • function_body: 函数体，函数中需要执行的代码语句块。
  • result_params_comma_separated: 函数返回值，Lua语言函数可以返回多个值，每个值以逗号隔开。

3. 变量

变量在使用前，需要在代码中进行声明，即创建该变量。变量的默认值均为 nil。

未使用 local 显式声明为局部变量的变量全部为全局变量，即使是语句块或函数内。

局部变量的作用域为从声明位置开始到所在语句块结束。

注册变量GI GR，作用域为整个工程，是一个长度为50000的数组

4. 脚本实现机制

4.1 虚拟机

通过调用c api 创建 luaL_newstate()，lua_State结构体代表一个Lua虚拟机，可同时创建多个，内部为单线程多实例实现，意味着各自创建的虚拟机栈相互隔离。

上位机每个文件对应一个虚拟机，嵌入式每个task对应一个虚拟机

4.2 文件管理

4.2.1 上位机

工程保存在 motionData/lua/ 路径下

运行时直接读取内存中的字符串

4.2.2 嵌入式

每个核静态申请TASK * 3个长度为[65535]的数组，文件根据文件名索引，覆盖同文件名文件

没有flash，开卡需要重新下载

ms每次下载时会保存MD5，运行时校验

4.2 函数注册

lua支持注册C函数或指针。Lua调用C 函数时，也使用了一个与C语言调用Lua 函数时相同类型的栈，C 函数从栈中获取参数，并将结果压入栈中。这个栈不是一个全局结构;每个函数都有其私有的局部栈。当Lua 调用一个C 函数时，第一个参数总是位于这个局部栈中索引为1的位置。即使一个C函数调用了Lua代码，而且Lua代码又再次调用了同一个（或其他）的C函数，这些调用每一次都只会看到本次调用自己的私有栈，其中索引为1的位置上就是第一个参数。

所有GT指令通过函数注册的方式调用

GI, GR, getIArray, getRArray, getAxisRef 通过注册为userdata使用

4.3 线程

由于裸核没有OS，通过lua协程以分时复用的方式模拟线程。

通过注册hook函数捕获lua运行状态，每30行字节码指令进入一次hook, 判断时间 > 100us时让出

无法做到实时，因而sleep函数需要通过周期实现

4.5 通讯

通过指令 GTN_SendPlcCommand 交互, 单次可交互470 word。

每一次背景循环会处理一条指令。

指令通过改变task的状态影响lua状态机的运行。

在hook函数中，lua状态机根据当前状态调用事件。

过于频繁的调用通讯指令会影响效率。

当固件不支持lua时，会被阻塞，没有返回值。

4.4 变量查询

4.4.1 全局变量

存于lua全局变量表_G中，以字符串为索引的hash表。作用域为当前虚拟机。

由于注册了大量指令以及全局变量，不便于批量查询，因而需要手动输入变量名进行索引。

4.4.2 局部变量

遍历查询当前堆栈以及外层非全局作用域的堆栈中所有局部变量。

嵌入式中由于协程让出保存上下文信息会影响堆栈信息，localvalue会存在部分缺失的现象

4.4.3 注册变量

额外注册的数组变量，作用域为整个工程。嵌入式则为当前核。

为兼容lua语法，下标索引也从1开始。GI, GR各为长度50000的数组。
